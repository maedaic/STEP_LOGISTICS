# 配送レイアウトシステム Ver1.1（STEP Studio）

紙の手書き積載レイアウトを置き換える「配送積載メモ」のデジタル版（MVP）。
インストール不要・オフライン動作。すべて実寸(mm)でデータを保持し、画面は縮尺変換して描画するだけ。

## 起動方法

- かんたん: `index.html` をブラウザ（Chrome推奨）でダブルクリックして開く
- 保存/読込を安定させたい場合: このフォルダで簡易サーバを起動してから開く
  ```
  python3 -m http.server 4599
  # → http://localhost:4599 を開く
  ```

## 使い方

1. 左上「＋商品アップロード」で写真/PDFを読込 → 右の商品一覧が生成（※OCRは現在スタブ）
2. 左のトラックをクリックして荷台を表示（起動時は0台）
   - STEP車両：クリックで表示／もう一度クリックで削除（ON/OFF・重複なし）
   - 業者4t：クリックのたびに 業者4t-1, -2, -3… と複数追加
3. 右の商品をクリック→荷台をクリック、またはドラッグ＆ドロップで配置
   - **当たり判定**：商品同士は重ねられません（重なる位置は赤枠・配置不可）／荷台外へは出せません
   - 配置済みの箱：ドラッグで移動 / ダブルクリックで90°回転 / ×で取消（残数は自動で戻る）
4. 残数を確認しながら「PDF出力」（A4横で印刷 → 現場で手書き追記）
5. 「保存」でJSON書き出し（＋自動でブラウザにも保存）、「開く」でJSON読込
   - 「自動保存 ON」で保存フォルダを一度選ぶと、以降 案件名.json へ自動上書き（Chrome/Edge・localhost/https）

## ファイル構成

- `index.html` — 画面構造
- `css/styles.css` — スタイル
- `js/data.js` — トラックマスター / 商品マスター（**寸法の調整はここを編集**）
- `js/app.js` — 状態管理・描画・配置・保存・PDF

## MVP実装済み（Ver1.1）

商品アップロード（UI）/ OCR（スタブ）/ 商品一覧生成 / トラック選択（ON/OFF・業者4t複数追加）/
実寸キャンバス / 商品配置 / **当たり判定（商品同士・荷台外）** / 残数管理 / 保存・自動保存・読込 / PDF出力

※ Ver1.0 からの変更: メモ欄を削除（現場は印刷して紙へ手書きするため）、起動時トラック0台、当たり判定を追加。

## 今回スタブ・対象外

- **OCR**: アップロードUIと結果反映の枠組みのみ。実エンジンは `js/app.js` の
  `runOcrStub()` を差し替えれば本番化できる（品番・数量を返す）。
- 対象外（SDS通り）: AI積載 / 積載率 / 荷重計算 / はみ出し判定 / 二段積み判定 /
  必要台数計算 / 配送費・人員計算 / 最適積載提案

## 注意（要確認データ）

`js/data.js` のトラック荷台内寸で `estimated: true` が付いたもの
（デュトロ③・小トラ・ボンゴ・業者4t）は暫定値です。実測値が分かり次第更新してください。
